1. Prerequisite

Easy Batch requires a Java 1.6+ runtime.

2. Use Easy Batch

2.1 Maven artifact

To use Easy Batch, you have to add easybatch-core-{{ site.version }}.jar to your application's classpath.

If you use maven, add the following dependency to your project's pom :

<dependency>
    <groupId>org.easybatch</groupId>
    <artifactId>easybatch-core</artifactId>
    <version>{{ site.version }}</version>
</dependency>

2.2 Maven quick start archetype

Easy Batch provides a maven archetype to get you started quickly. This archetype generates a skeleton project for you with the right dependency to the framework's core. In order to generate a quick start project, just run the following command:

$>mvn archetype:generate \
    -DarchetypeGroupId=org.easybatch \
    -DarchetypeArtifactId=easybatch-archetype \
    -DarchetypeVersion={{ site.version }}

You will be asked for your project's groupId, artifactId, version and base package. Once finished, you will get a fully functional sample batch application that you can run with the following command:

$>mvn compile exec:java -Dexec.mainClass=${your.base.package}.App

3. Build Easy Batch from source

3.1 Get source code

Easy Batch uses Git as source control management tool and is hosted at GitHub. To download Easy Batch source code, clone it's git repository as follows:

$>git clone https://github.com/benas/easy-batch.git

You can also get the source code in a zip file from the home page of the project at GitHub.

3.2 Build with Maven

Easy Batch uses maven as build tool. To build Easy Batch binaries, use the following command from the root directory of the project:

$>cd easy-batch
$>mvn install
